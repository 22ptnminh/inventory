ENTITY CORE — Tầng tiếp nhận & Kết nối Client
ENTITY CORE ENGINE • Đầu mối kết nối trực tiếp API với CLIENT
• Xác thực đối tác (API Key, signature) & Nhận/Trả kết quả Request
• Cập nhật trạng thái đơn hàng cuối cùng (Final Order Status)
• Tạo Transaction (source of truth) · idempotency via correlation_id
🔐 Auth & ValidationAPI Key · Rate limit · Schema validation
📝 Transaction LogTạo TXN · correlation_id · flow_direction: CHARGE | REDEEM
🔄 Status CallbackWebhook trả kết quả về partner khi hoàn tất
↓Route sang UrCard Core xử lý
5
URCARD CORE — Tầng xử lý giao dịch
🎯 Campaign Processing • Quản lý Campaign ID, Client Budget (PO/SubPO), Giftset, Expiry
• Tính Doanh thu dựa trên Selling Price (Markup)
💳 Thẻ lượt / Điểm Đơn vị thanh toán dùng để mua quà ở tầng dưới
• TOPUP_BALANCE: cộng điểm thẳng · TOPUP_CATALOG: cộng lượt
• DEBIT: trừ khi user đổi quà
📦 Order Management (POM) Tạo Order khi PAYMENT · idempotency · PENDING→PROCESSING→COMPLETED/FAILED
⚑ Deduct-first: Trừ điểm/lượt TRƯỚC khi gọi Rewards Core · Nếu fail → hoàn lại (rollback entry)
Quà lượt/điểm (TOPUP_BALANCE / TOPUP_CATALOG): xử lý tại đây, không đẩy xuống Rewards Core
↓Chuyển tiếp request mua quà xuống Rewards Core
6
REWARDS CORE — Tầng định hướng loại quà
🔀 Điều hướng loại quàDetect từ giftDetailId → output_type → route đúng handler
📅 Quà Booking đối tácDragonPass · Premium Service · Fasttrack · Đặt chỗ realtime
💳 Topup đối tácTopup trực tiếp · VNA · Viettel · Ví điện tử
🎫 E-Voucher đối tácVoucher tiền mặt đối tác sinh mã · Thẻ điện thoại · Items · Deal · Lockitem SKU
🎟 E-Voucher UrBoxVoucher tiền mặt UrBox tự sinh (CashVoucher)
📦 Quà vật lýShip hàng GHTK/GHN · Trừ tồn kho SKU
🔄 Quà lượt/điểmChỉ topup điểm hoặc lượt cho thẻ · Xử lý tại UrCard Core, không xuống đây
🔀 Điều hướng cách lấy code
Booking (realtime)?→Gọi API đối tác ngay (DragonPass...)
Topup đối tác?→Gọi API nhà mạng · Không sinh mã
E-Voucher đối tác?→Lấy từ kho (Redis/MySQL) hoặc gọi API
E-Voucher UrBox?→UrBox tự sinh mã theo prefix+pattern
Quà vật lý?→Không sinh mã · Trừ SKU · Đẩy đơn ship
↓Gọi API / lấy mã từ Merchant

MERCHANT CONNECTOR LAYER — Kết nối Merchant
MERCHANT CONNECTOR ENGINE Chuyên trách giao tiếp, gọi API/Webhook kết nối với Merchant bên ngoài để lấy mã Code thời gian thực (Real-time Get Code)
⚡ API RealtimeGọi đồng bộ lấy mã ngay · DragonPass, Grab, Shopee...
📋 API Lô / PONhập mã theo lô từ Merchant · Garena, CGV, Vincom...

↓Lưu mã / Trừ stock
8
INVENTORY LAYER — Tầng kho & Giá vốn
🏭 QUẢN LÝ KHO • ID Kho · Code stock · COST (Giá vốn / COGS)
• Tồn kho realtime · Low-stock alert
• Xuất kho mã lưu sẵn (In-house Stock Code)
🗄 KHO CODE (codex) • Mã code, Giá trị sử dụng, Serial, PIN
• Trạng thái: 111=READY · 210=ISSUED · 300=USED
• transaction_id → link về Gateway TXN
📉 Trừ stock Redis LPOP (atomic) · MySQL update process
• WhoExport=0: nhập sẵn · =1: UB tự sinh · =2: API realtime
↓Lấy mã từ Provider / Aggregator
